Syntactically Awesome Style Sheets (Sass) is a style sheet language initially designed by Hampton Catlin and developed by Natalie Weizenbaum. After its initial versions, Weizenbaum and Chris Eppstein have continued to extend Sass with SassScript, a simple scripting language used in Sass files.

Here is basic HTML file

hello.html

<!DOCTYPE html>
<html>
    <head>
        <title> Webpage Title </title>
    </head>
    <body>
        <p> Hello, World! </p>
    </body>
</html>

And also a SASS style file

sass_style.scss

/*basic variable*/
$color = red;

p {
    background-color: $color;
}

The SASS code cannot be directly used as browsers will not be able to interpret the same as required. So the SASS code is converted into CSS which the browsers can interpret. The below command is used to do this conversion.

sass sassFileName.scss newCssFlieName.css

This manual conversion is not required every time a change is made to the initial SASS file. The generated CSS file can be made to track changes in the original SASS file and make necessary changes automatically.

sass --watch sassFileName.scss:newCssFlieName.css

SASS also allows nesting of selectors to keep the code organized and simple.

div {
    font-size: 20px;

    p {
        color: blue;
    }

    li {
        color: green;
    }
}

Sass Inheritance, @extend is used to share a set of CSS properties from one selector to another. The @extend feature of Sass allows classes to share a set of properties with one another. In complicated CSS where many classes are put together, duplication of properties may occur.

%mainSegment {
    font-family: sans-serif;
    font-size: 18px;
    font-weight: bold;
    border: 1px solid black;
    padding: 20px;
    margin: 20px;
}

.className {
    @extend %mainSegment;
    background-color: green;
}
